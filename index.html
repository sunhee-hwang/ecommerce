1. 프로젝트 개요 및 문제 정의

이 프로젝트는 차세대 E-commerce 플랫폼을 구축하여 기존 서비스의 성능과 사용자 경험을 혁신하는 것을 목표로 했습니다.

기존 커머스 사이트의 복잡한 UI와 느린 로딩 속도(평균 TBT 3초 이상)는 높은 사용자 이탈률의 주된 원인이었습니다. 이를 개선하여 사용자 경험을 최적화하고, 최종적으로 구매 전환율을 높이고자 했습니다.

2. 아키텍처 및 설계

문제 해결을 위해 Next.js의 SSR/ISR을 핵심 전략으로 채택했습니다. 데이터베이스는 PostgreSQL을 사용했으며, Node.js 기반의 REST API를 설계했습니다. 아래는 시스템의 전체 아키텍처 다이어그램입니다.

시스템 아키텍처

3. 핵심 기능 및 구현

A. 서버 사이드 렌더링 (SSR)을 통한 성능 최적화

Next.js의 `getServerSideProps`를 사용해 상품 상세 페이지를 서버에서 렌더링했습니다. 이를 통해 초기 로딩 시점(TBT)을 40% 단축하고 검색 엔진 최적화(SEO)를 개선했습니다.

B. React Query를 활용한 상태 관리

클라이언트 사이드의 비동기 데이터를 React Query로 관리했습니다. `stale-while-revalidate` 캐싱 전략을 통해 불필요한 API 호출을 획기적으로 줄이고 사용자 경험을 향상시켰습니다.

4. 나의 역할 및 기여

팀 내 프론트엔드 개발자로서 다음 역할을 담당했습니다.

  • Next.js 기반 프로젝트 아키텍처 설계 및 초기 세팅
  • React Query 도입 및 전역 상태 관리 전략 수립
  • 상품 상세 페이지 및 장바구니 기능 SSR/CSR 구현
  • 팀 내 코드 리뷰 및 Git 브랜치 전략(Git-flow) 관리

5. 결과 및 회고

프로젝트를 통해 다음과 같은 정량적/정성적 성과를 달성했습니다. (강의 5번 - 성과 수치화)

  • Lighthouse 점수 30점 향상 (Performance 55 -> 85)
  • 초기 로딩 속도(TBT) 40% 단축
  • TypeScript를 통한 코드 안정성 확보 및 배포 전 버그 20% 감소

배운 점: SSR의 동작 원리와 React Query의 캐시 전략을 깊이 이해할 수 있었습니다.

개선점 (면접 대비): (강의 5번 - 다시 만든다면?) 향후 Incremental Static Regeneration (ISR)을 도입하여 빌드 시간을 단축하고, Jest와 React Testing Library를 활용한 테스트 코드를 작성하여 코드 안정성을 더욱 높이고 싶습니다.
